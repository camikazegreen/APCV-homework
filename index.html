Fun Font Experiments!

  1. My first step was to look for interesting fonts on DaFont. There were lots of interesting fonts to choose from, but with the small sample of fonts that I tried to use I found their fonts to be unreliable. The heading on this page is rendering El Rio Lobo incorrectly for example, and is displaying errors in the console for "bad table directory". This paragraph is rendered using a groovy font called Happiness Mood.

    Since the DaFont fonts that I downloaded only came with a TrueType file when I downloaded them, I looked up how to render local TrueType fonts on Stack Overflow. I moved the ".ttf" files into my fonts folder and added them into my project using the @font-face declaration.

    @font-face {
    font-family: 'happiness_mood';
    src:
    url('fonts/Happiness_Mood.ttf') format('truetype');
    }

    Next, I defined the css to actually use this new font-family:

    p.happiness {
    font-family: "happiness_mood", sans-serif;
    font-weight: 500;
    font-style: normal;
    }

    This seemed to work fine, except for the errors mentioned for the El Rio font. However, most of the comments on StackOverflow were recommending the use of .woff or .otf files instead of or in addition to .ttf, and the fonts that I downloaded on DaFont only provided a .ttf file.

  2. In order to try to find fonts with the recommended file types from StackOverflow, I switched over to Google Fonts and found this cool font called Single Day. Unfortunately, it also only provided a TrueType variant and I was not able to find other font downloads that provided .woff or .otf file downloads. I added this font using the same method as the early font, and it worked flawlessly.

  3. Then I switched over to using Adobe Typekit since we have free access to this service as students at the University of Arizona. Using Adobe Typekit was easy and I didn't need to refer to any outside documentation, since the appropriate links and css declarations were provided when choosing a font:

    p.nudgewink { font-family: "p22-nudgewink-pro", sans-serif; font-weight: 300; font-style: normal; font-size: large;

    The only downside to Adobe Typekit, is that there is a noticeable lag when loading the page before it loads the font from Adobe.
